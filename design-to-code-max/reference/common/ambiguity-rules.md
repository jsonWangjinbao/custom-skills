# 歧义检测规则 & 提问模板

## 基础规则（始终生效，无论是否有截图）

### 自动决策

满足以下任一条件，直接处理，记录到 STYLE-REVIEW.md：

1. **CSS 变量命中映射表**：`var(--Gray-7)` 在 `reference/rn/token-map.json` 中找到对应 theme key → 使用 `theme['color-background-page']`，具体色值以 `token-map.json` 的 `themeObject` 为准
2. **图标命中映射表**：HTML/截图中的图标语义在 `reference/rn/icon-map.md` 或 `iconfontGlyphMap` 中找到对应 key → 使用 `<XlbIcon name="Key" ... />`
3. **真实图片路径存在**：`src="./asset/xxx.png"` / `src="./asset/xxx.jpg"` 在 assets 目录中存在 → 使用 `<Image source={require('...')} />` / `<XlbImage source={...} />`
4. **SVG 文件路径存在**：`src="./asset/icons/xxx.svg"` 在 assets 目录中存在 → **不直接 require**，优先映射为 `XlbIcon`；若无法映射则标记「图标无映射」风险
5. **flex 属性标准**：值在 CSS flexbox 标准范围内 → 直接翻译
6. **尺寸/颜色为明确数值**：HTML 中声明了具体 px 值或 HEX 色值 → 采用后必须映射为 token（颜色 → theme key，间距/圆角/字号 → SPACE/BORDER/FONT）

### 必须提问

#### 规则 1：CSS 变量未解析

**条件**：CSS 变量不在映射表中，且（无截图 或 截图也无法推断）
**提问模板**：

```
⚠️ 无法解析的颜色变量：
- 变量名：var(--{Name})
- 映射表中未找到
- 请提供该变量对应的 theme key；若项目无对应 token，请确认是否新增 token 或直接写死 hex（需记录风险）
```

#### 规则 2：字体平台兼容

**条件**：设计稿字体在目标平台（特别是 RN Android）上默认不可用
**提问模板**：

```
⚠️ 字体兼容性：
- 设计稿使用「{字体名}」
- 在 {平台} 上默认不支持此字体
- 选项 1：替换为系统字体
- 选项 2：保留原名，后续自行配置字体文件
- 请选择处理方式
```

#### 规则 3：复杂定位歧义

**条件**：HTML 使用 `position: absolute` + `calc()` 表达式，或定位元素与父容器关系不明确
**提问模板**：

```
⚠️ 复杂定位需要确认：
- HTML 定位表达式：{calc() 或定位值}
- 计算后的实际位置：{推算结果}
- 这个定位是否符合预期？请确认元素的实际位置。
```

#### 规则 4：元素语义不明确

**条件**：无法从 HTML 结构判断元素的用途
**提问模板**：

```
⚠️ 元素语义需要确认：
- 元素：{位置描述 + 内容摘要}
- HTML 标签：{div/span/img}，尺寸 {w}x{h}px
- 这个元素是装饰性元素还是功能组件？
```

#### 规则 5：图标无法映射到 icon-rn

**条件**：HTML/截图中的图标在 `reference/rn/icon-map.md` 和 `iconfontGlyphMap` 中均找不到对应 key
**默认处理**：使用语义最接近的候选图标替代，在代码注释中说明降级原因，并在 `ui-audit.md` 标记「图标降级：{原语义} → {替代 name}」

**必须提问**：只有当替代图标与原始语义差异过大（如用 `File` 代替「人」图标）且用户明确要求精确还原时，才使用 `AskUserQuestion`：

```
⚠️ 图标无法精确映射：
- 图标语义/位置：{描述}
- 候选替代图标：{最接近的 1-3 个 name}
- 默认方案：使用 {候选 name} 替代（存在视觉差异）
- 是否需要推动 UI 补充精确图标？
```

#### 规则 6：图片与图标边界模糊

**条件**：HTML 元素是 `<img>` 或截图元素无法判断是图标还是真实图片
**提问模板**：

```
⚠️ 图片/图标边界需要确认：
- 元素：{位置描述}
- HTML：{<img src="..."> 或截图区域}
- 尺寸：{w}x{h}px
- 内容：{照片 / 插画 / 矢量小图标 / 文字标识}
- 这是图标（用 XlbIcon）还是真实图片（用 Image/XlbImage）？
```

## 截图交叉验证规则（仅当用户提供了设计截图时生效）

调用 `image-recognize` skill 分析截图后，额外执行以下规则。

### 自动修正（不提问）

| 条件                                            | 处理                                                 |
| ----------------------------------------------- | ---------------------------------------------------- |
| CSS 变量在映射表命中，截图颜色与映射值偏差 ≤ 3% | 以 token 为准                                        |
| HTML 数值与截图偏差 ≤ 5px                       | 以 token 为准（先找最接近的 SPACE/BORDER/FONT 常量） |

### 必须提问

#### 截图规则 1：尺寸偏差

**条件**：截图推算的关键尺寸与 HTML 声明值偏差 > 5px
**提问模板**：

```
⚠️ 尺寸差异：
- 位置：{元素描述}
- HTML：{属性} = {X}px
- 截图推算：约 {Y}px（偏差 {Δ}px）
- 请选择最接近的 token：{候选 SPACE/BORDER/FONT token}，或确认其他值
```

#### 截图规则 2：缺失/多余元素

**条件**：截图中存在 HTML 未包含的视觉元素，或 HTML 中存在截图上看不到的元素
**提问模板**：

```
⚠️ 元素差异：
- 截图中有但 HTML 中没有：{位置描述}
- 请确认这是否为设计稿的一部分？
```

#### 截图规则 3：层级矛盾

**条件**：HTML 的 DOM 顺序/z-index 与截图视觉层级不一致
**提问模板**：

```
⚠️ 层级差异：
- HTML：{元素A} 在 {元素B} {上方/下方}
- 截图：层级关系与 HTML 不符
- 请确认正确的层级顺序。
```

### 偏差计算公式

- **尺寸偏差**：`|html_value - screenshot_estimated_value|`
- **颜色偏差**：`max(|R1-R2|, |G1-G2|, |B1-B2|)`，> 8 时视为显著差异

## 样式检查清单

生成代码前逐项检查，每项标注 PASS / FIXED / QUESTION：

```
□ 容器 padding/gap ── 已映射为 SPACE.* / BORDER.*
□ 子元素宽度 ── 固定值 vs 内容撑开
□ 文本字号、行高、字重 ── 已映射为 FONT.*
□ 颜色（背景、文字、边框）── 已解析为 theme['xxx']
□ 圆角值 ── 已映射为 BORDER.*
□ 阴影参数 ── 已映射为 SHADOW.* 或显式 shadow token
□ 图片/SVG 尺寸和位置 ── 与 HTML 一致
□ 图标已映射为 XlbIcon 且 name 在 iconfontGlyphMap 中 ── 通过对照确认（缺失时允许降级替代）
□ 真实图片已使用 Image / XlbImage + require ── 路径正确
□ 无直接 require SVG 文件 ── 通过 Grep 扫描确认
□ z-index 层级顺序 ── 与 HTML 一致
□ CSS 变量已全部解析 ── 映射为 theme key 或记录风险
□ 无硬编码 hex / rgba / rgb（含 constants.ts 等配置文件）── 通过 Grep 扫描确认
□ 无 emoji / 文字符号代替图标 ── 通过 Grep 扫描确认
□ 无 icon_ 前缀的错误图标名 ── 通过 Grep 扫描确认
```

- 无截图模式：所有项以 HTML 为准，FIXED 标记不适用
- 有截图模式：与截图不一致的项标注 FIXED + 修正后的 token 引用
