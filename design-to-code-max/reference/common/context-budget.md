# 上下文预算规则

> 控制各阶段文档体积与阶段间上下文传递量，防止流程后段上下文膨胀稀释关键证据。
> 版本: 1.0（2026-07-23，借鉴 xlb-plugin sdd-execution-policy 上下文预算机制）

---

## 1. 文档体积上限

| 文档             | 上限        | 超限处理                                                             |
| ---------------- | ----------- | -------------------------------------------------------------------- |
| `features.md`    | 8,000 字符  | 功能点表只保留 ID/描述/验收标准列，细节移入附录节                    |
| `api-spec.md`    | 12,000 字符 | 每个接口只保留一张字段映射表，响应示例截断至关键字段                 |
| `ui-audit.md`    | 12,000 字符 | 三要素表只保留与 token 有差异的行，全等行合并为「其余属性全等 ✅」   |
| `tech-design.md` | 8,000 字符  | 只保留决策结论与理由，推导过程压缩为一行                             |
| `execution.md`   | 10,000 字符 | 已完成分组压缩为一行摘要（分组号 + 文件 + 结果），进行中分组保持完整 |

**压缩红线**（任何情况下不得删除）：

- 验收标准与待确认项
- 证据索引（文件路径、parsed-styles JSON 路径、截图路径）
- 组件选择决策表的「差异 + 补偿方案」列
- api-spec 中标注 ✅ 的字段映射行

**解析产物豁免**：`parsed-styles/*.json` 是 build 阶段样式数据的唯一来源，属于数据产物而非叙述文档，**不参与压缩、不得删改**（保护性约束 1）。

---

## 2. 阶段间上下文传递

后续阶段**默认不整篇回读**前序文档，按下表定点读取：

| 消费阶段          | 读取内容                                                                      | 不读                                            |
| ----------------- | ----------------------------------------------------------------------------- | ----------------------------------------------- |
| api-spec          | features.md 的「接口清单表」一节                                              | features.md 全文                                |
| audit             | features.md 功能点表 + api-spec.md 接口清单                                   | 需求原始材料全文（已在 collect-materials 落盘） |
| design            | ui-audit.md 的「组件选择决策表」+「关键样式规格」章节路径                     | ui-audit.md 全文、原始 HTML                     |
| build（每个分组） | execution.md 当前分组 + 该分组涉及的 parsed-styles JSON（定点 Read 单个文件） | 其他分组的 parsed JSON、ui-audit.md 全文        |
| verify            | execution.md 分组摘要 + features.md 验收标准 + git diff                       | build 过程的中间叙述                            |

只有当定点读取的内容缺少关键证据时，才允许回读全文，并在阶段日志中记录回读原因。

---

## 3. 提问与材料收集预算

- 需求原始材料（PRD 全文、后端设计全文）在 collect-materials 阶段落盘保存路径，后续阶段引用路径而非粘贴全文。
- 单个参考文档超过 12,000 字符时，先读目录/大纲，再定点读取最多 4 个相关章节。
- 截图逐张 Read 是强制双源交叉（保护性约束 2），**不受章节数限制**，但每张截图的提取结论只写 `screenshotFindings` 结构化数组，不逐字描述画面。

---

## 4. 与状态机的联动

- 文档超限压缩后，在对应 `phaseOutputs.<phase>` 中追加 `compressed: true` 标记，verify 阶段据此复核压缩红线未被触碰。
- 中断恢复时按本规则定点读取，禁止为「恢复上下文」整篇回读所有文档。
